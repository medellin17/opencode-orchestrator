# Верификация реализации

## Проходы

### 1. Соответствие плану (Проход F)
Сравнить реализованный код с исходным планом.
*   Убедиться, что все запланированные изменения реализованы в полном объёме.
*   Выявить код, отсутствующий в плане, и определить цели его добавления.
*   Выяви расхождения/регрессии/проблемы безопасности/производительности. Если не найдено — явно напиши: «[Категория]: не выявлено».
*   При полном соответствии вернуть фразу: "Реализация соответствует плану".

### 2. Регрессионный анализ (Проход G)
Проверить стабильность существующей системы после внедрения нового кода.
*   Проанализировать связи измененного кода с остальной системой.
*   Выяви расхождения/регрессии/проблемы безопасности/производительности. Если не найдено — явно напиши: «[Категория]: не выявлено».
*   При отсутствии рисков детально обосновать отсутствие регрессионного влияния.

### 3. Финальная проверка качества (Проход H)
Провести аудит безопасности, производительности и надежности кода.
*   Выяви расхождения/регрессии/проблемы безопасности/производительности. Если не найдено — явно напиши: «[Категория]: не выявлено».

---

## Итоговый чеклист
Перед завершением верификации подтвердить выполнение следующих пунктов:
- [ ] Проход F (соответствие плану) выполнен и задокументирован.
- [ ] Проход G (отсутствие регрессий) выполнен с разбором связей.
- [ ] Проход H (финальная безопасность/производительность/ошибки) пройден.
- [ ] Все критические (`[Critical]`) и мажорные (`[Major]`) проблемы устранены.
