# Создание плана разработки

## Промпт

При получении задачи разработать структурированный план реализации. Разделить работу на логические части (карточки). Каждая карточка должна содержать 4 обязательные секции:

1. **Зачем?** (Контекст / Проблема)
   - Описать причину реализации и решаемую проблему.
   - Для багфикса: указать критичность и причину возникновения.
   - Для фичи: описать, чего не хватает системе на текущий момент.
2. **Что?** (Ожидаемый результат / Поведение)
   - Описать ожидаемое поведение системы после реализации.
   - Привести конкретные сценарии (пользователь делает X → система отвечает Y).
   - Четко очертить границы задачи: что входит в текущие работы, а что нет.
3. **Как?** (Шаги реализации)
   - Разбить реализацию на последовательные шаги.
   - Указать создаваемые или изменяемые файлы.
   - Описать зависимости от других частей плана и технические решения.
4. **Как проверить?** (Критерии готовности)
   - Описать проверки для подтверждения корректности (функциональные, интеграционные, визуальные).
   - Привести edge cases, которые должны быть проверены.

Для каждого этапа выяви потенциальные риски и допущения. Если этап не имеет очевидных рисков — отметь неявные допущения.

> [!IMPORTANT]
> Не писать код в процессе генерации. Создавать только структуру плана.

Сохранить план в `PLAN.md` в корне проекта.
