# Мульти-агентный сканер системы

Запустить параллельный аудит системы или крупного модуля с использованием специализированных суб-агентов.

> [!IMPORTANT]
> Запрещено проводить весь аудит самостоятельно в одном контексте. Обязательно распределить задачи по отдельным суб-агентам для каждого домена.

## Промпт

1.  **Разделение на домены**: Разбить целевую систему на 2–5 специализированных технологических или функциональных доменов.
2.  **Запуск суб-агентов**: Для каждого домена инициализировать отдельного суб-агента со следующей инструкцией:
    *   Провести глубокий анализ кода/архитектуры в рамках выделенного домена.
    *   Найти минимум 2 критичные проблемы с точным указанием файла, строки и обоснования.
3.  **Агрегация результатов**:
    *   Собрать отчеты всех суб-агентов в единый реестр.
    *   Очистить список от дубликатов и надуманных проблем.
    *   Классифицировать проблемы по критичности (`[Critical]`, `[Major]`, `[Minor]`).
    *   Сформировать итоговый **ТОП-5 угроз**, не допустимых для выпуска в релиз.

## Примеры разбиения по доменам

### Для веб-приложений:
1.  **Базы данных и миграции**: схемы, индексы, связи, N+1, транзакции.
2.  **Авторизация и сессии**: JWT, cookie, права доступа, CORS.
3.  **API и контракты**: форматы ответов, валидация входящих данных, обработка ошибок.
4.  **Бизнес-логика**: консистентность, race conditions, граничные случаи.
5.  **Клиент и UI**: рендеринг, стейт-менеджмент, offline-режим.

### Для мобильных приложений:
1.  **Сетевой слой**: API клиент, политики повторных попыток (retry), таймауты.
2.  **Роутинг и навигация**: deep-links, пуш-уведомления, инициализация при старте.
3.  **Локальное хранилище**: кэширование, синхронизация данных, offline-работа.
4.  **Авторизация**: жизненный цикл сессии, обновление токенов (refresh), logout-сценарии.
5.  **Интерфейс и производительность**: рендеринг списков, плавность анимации, утечки памяти.

These are examples — choose domains that match the actual system under audit.

Launch sub-agents using the `task` tool. Each sub-agent prompt must include: Goal, Context (copy-pasted domain-specific code/docs), Deliverable, Constraints.

## Output Template

For each finding, use the following format:

- **Problem description**: what was found.
- **Severity**: `[Critical] / [Major] / [Minor]`.
- **File/line**: exact location in code.
- **Recommendation**: concrete fix or mitigation.

**Результат сохранить в**: `.opencode/context/multi-agent-scan-report.md`
