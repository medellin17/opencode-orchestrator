# Глубокий аудит архитектуры (Deep Audit)

Провести всесторонний архитектурный и системный анализ кодовой базы или плана реализации с использованием 3 суб-агентов.

> [!IMPORTANT]
> Обязательно делегировать аудит суб-агентам по указанным ниже направлениям. Не пытаться выполнять весь глубокий аудит в основном контексте.

## Промпт

### 1. Запуск суб-агентов

Launch sub-agents using the `task` tool with the prompts below.

Инициализировать 3 специализированных суб-агентов со следующими задачами:

*   **Суб-агент 1: Аудит допущений (Assumption Audit)**
    Выявить не менее 10 неявных предположений автора о работе системы (например, "база всегда доступна", "сеть не имеет задержек").
    Для каждого допущения описать:
    1.  Суть предположения (одно предложение).
    2.  Место в коде, где оно неявно заложено.
    3.  Сценарий отказа (что произойдет, если предположение неверно).
    4.  Наличие fallback-механизма (если отсутствует — зафиксировать дефект).

*   **Суб-агент 2: Проверка инвариантов (Invariant-Based Analysis)**
    Определить 5 критических бизнес- и системных инвариантов (правил, которые должны оставаться истинными в любой момент времени).
    Для каждого инварианта описать:
    1.  Четкую формулировку инварианта.
    2.  Место в коде, отвечающее за его поддержание.
    3.  Сценарий, при котором инвариант может быть нарушен.
    4.  Способ обнаружения нарушения (метрики, алерты).
    5.  Последствия нарушения для бизнеса/системы.

*   **Суб-агент 3: Временной анализ (Temporal Analysis)**
    Оценить эволюцию системы и риски в трех временных масштабах:
    *   **T+0** (Немедленный запуск): проблемы первых 100 пользователей, мгновенные сбои.
    *   **T+1 мес** (Рост нагрузки в 10 раз): узкие места (bottlenecks), race conditions.
    *   **T+6 мес** (Эволюция продукта): критический техдолг, компоненты под полный рефакторинг.
    Для каждого горизонта привести минимум 2 конкретные проблемы со ссылками на код.

### 2. Сведение результатов
После завершения работы всех суб-агентов:
*   Объединить результаты их работы в единый реестр.
*   Удалить дубликаты и отфильтровать малозначимые замечания.
*   Классифицировать проблемы по критичности (`[Critical]`, `[Major]`, `[Minor]`).
*   Выделить **ТОП-3 системные угрозы** архитектуры.

## Output Format

**Результат сохранить в**: `.opencode/context/deep-audit-report.md`

### Report Template

```markdown
# Deep Audit Report

## Summary
[Overall assessment of the architecture]

## Findings

### Sub-agent 1: Assumption Audit
- Finding: ...
- Severity: [Critical/Major/Minor]
- Recommendation: ...

### Sub-agent 2: Invariant-Based Analysis
- Finding: ...
- Severity: [Critical/Major/Minor]
- Recommendation: ...

### Sub-agent 3: Temporal Analysis
- Finding: ...
- Severity: [Critical/Major/Minor]
- Recommendation: ...

## Recommendations
[Top systemic threats and suggested mitigations]
```
